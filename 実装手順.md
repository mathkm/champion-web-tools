# ampereVisualizer 実装手順書(Sonnet 5 向け作業指示)

本書は [要件定義.md](要件定義.md) に基づく実装を、単独で完遂するための手順書。
この手順書だけで作業できるよう、データ仕様・実装詳細・検証手順をすべて含む。

## 0. ゴール

`C:\dev\ampereVisualizer\index.html` という **単体HTMLファイル1つ** を作成する。
ブラウザで開き、CSVをドラッグ&ドロップすると、電流2チャンネルの時系列グラフと
統計サマリが表示されるダッシュボードを実装する。**サーバー不要・ビルド不要。**

完成条件:
- ブラウザで `index.html` を開くだけで動く(Chrome/Edge)。
- CSVを複数まとめてD&Dすると、受信日時順に連結して1本の時系列で表示される。
- ズーム/パン、チャンネル表示切替、表示期間に連動した統計が動作する。

## 1. 技術スタック(確定)

- ライブラリは **CDN読み込み** を基本とする(初回のみネット接続が必要)。
  - CSVパース: **PapaParse 5.x** … `https://cdn.jsdelivr.net/npm/papaparse@5.4.1/papaparse.min.js`
  - 描画: **Plotly.js 2.x** … `https://cdn.plot.ly/plotly-2.35.2.min.js`
- フレームワーク不使用。素のHTML + CSS + JavaScript(1ファイルに全部書く)。
- 文字コードは UTF-8 で保存し、`<meta charset="utf-8">` を付ける。
- (オプション)完全オフライン化が必要になったら、上記2ライブラリをローカルに
  ダウンロードして相対パス参照に差し替える。今回は実装しないでよい。

## 2. 入力データ仕様(厳守。ここを間違えるとパースが壊れる)

対象CSVは MQTT ロガー出力。サンプル1行:

```
2026/07/01 16:43:16.847,test/volt0,"{""is_ntp_synced"":false,""timestamp"":6989,""ch0_A"":1.587011,""ch1_A"":1.533177,""uptime_s"":6,""rssi"":-51}"
```

### 2.1 CSV構造
- 1行目はヘッダー: `受信日時,トピック,ペイロード`
- **先頭にUTF-8 BOM (`﻿`) が付く。** PapaParse は通常吸収するが、
  ヘッダー名比較時は BOM 除去を意識すること(ヘッダー名では判定せず列インデックスで
  扱うのが安全)。
- 列は3つ。第0列=受信日時、第1列=トピック、第2列=ペイロード(JSON文字列)。
- 第2列はダブルクォートで囲まれ、内部の `"` は `""` にエスケープされている。
  **PapaParse に生CSVを渡せばこのエスケープは自動解除される**ので、自前で
  `replace` しないこと(二重処理でバグる)。PapaParse 通過後の第2列は正規の
  JSON文字列 `{"is_ntp_synced":false,...}` になっている。

### 2.2 受信日時のパース
- 形式: `YYYY/MM/DD HH:MM:SS.mmm`(例 `2026/07/01 16:43:16.847`、ミリ秒3桁)。
- `new Date("2026/07/01 16:43:16.847")` は環境依存。**必ず自前でパースする**:
  正規表現 `/^(\d{4})\/(\d{2})\/(\d{2}) (\d{2}):(\d{2}):(\d{2})\.(\d{3})$/`
  で分解し、`new Date(y, mo-1, d, h, mi, s, ms)` を生成する。
- パース失敗行はスキップし、スキップ件数を数えておく(後で通知表示)。

### 2.3 ペイロードのフィールド
| キー | 型 | 用途 |
|---|---|---|
| `ch0_A` | number | 電流CH0 [A]。グラフ・統計の主対象 |
| `ch1_A` | number | 電流CH1 [A]。グラフ・統計の主対象 |
| `rssi` | number | WiFi強度 [dBm]。補助表示(任意) |
| `is_ntp_synced` | bool | NTP同期。補助表示(任意) |
| `timestamp`/`uptime_s` | number | 今回は未使用でよい |

- `JSON.parse` に失敗した行、`ch0_A`/`ch1_A` が数値でない行はスキップ(件数カウント)。

### 2.4 実測レンジ(検証・軸設定の目安)
- `ch0_A`: 0.245〜46.97、`ch1_A`: 0.249〜43.40、`rssi`: -80〜-48。
- サンプリング間隔 約1.4秒。1ファイル約1.5万行、5ファイルで約7万行規模。

## 3. データ処理仕様

### 3.1 読み込みフロー
1. D&D または `<input type="file" multiple accept=".csv">` で File を複数受け取る。
2. 各 File を `FileReader.readAsText(file, 'UTF-8')` で読む(または PapaParse に
   File を直接渡す。`Papa.parse(file, {header:false, skipEmptyLines:true, complete})`)。
3. 各行を `{ t: Date, ch0: number, ch1: number, rssi: number, ntp: bool, file: name }`
   の**レコード**に変換。ヘッダー行(第0列が `受信日時` を含む/日時パース失敗)は除外。
4. 全ファイルのレコードを1つの配列に集約。
5. **`t` の昇順でソート**(連結の肝)。同一時刻の重複は許容(または完全重複行は除去)。
6. ソート済み配列を「現在のデータセット」として保持し、再描画する。
7. 追加でファイルがD&Dされたら、既存に**追記マージして再ソート**(置き換えでなく累積)。
   ※「クリア」ボタンで全消去できるようにする。

### 3.2 ダウンサンプリング(描画性能対策)
- 表示点数の上限を定数 `MAX_POINTS = 5000` とする。
- レコード数が `MAX_POINTS` を超える場合、`Math.ceil(N / MAX_POINTS)` を間引き係数
  `step` とし、`step` 個ごとに1点抽出して描画用配列を作る(単純間引きでよい)。
- **統計は間引き前の全データで計算する**(描画だけ間引く)。
- Plotly のトレースは `type: 'scattergl'`(WebGL)を使い大量点でも軽快にする。

### 3.3 統計計算(表示期間連動)
「表示期間」= Plotly のX軸レンジ(初期は全期間)。以下をCH0/CH1それぞれ算出:
- 最大 (max)、最小 (min)、平均 (mean)、データ点数。
- **積算 A·h**: 台形則で電流を時間積分する。
  隣接2点 `i, i+1` について `dt_h = (t[i+1]-t[i]) / 3600000`(ms→時間)、
  `(ch[i]+ch[i+1])/2 * dt_h` を総和。単位は A·h。
  ※ dt が異常に大きい区間(例: ファイル境界で数時間空く)を積算に含めると過大に
  なる。**`dt` が閾値(例 60秒)を超える区間は積算からスキップ**する。
- 表示期間はデフォルト全期間。Plotly の `plotly_relayout` イベントで
  X軸レンジ変更を検知し、その範囲のレコードだけで統計を再計算して数値を更新する。

## 4. 画面レイアウト

縦積みのシンプルな1カラム構成でよい。

```
┌───────────────────────────────────────────┐
│ ヘッダー: タイトル「ampereVisualizer」        │
├───────────────────────────────────────────┤
│ [ドロップゾーン]                              │
│  「CSVをここにドラッグ&ドロップ / クリックで選択」 │
│  読み込み済みファイル一覧・件数・スキップ件数・[クリア]│
├───────────────────────────────────────────┤
│ [統計サマリ] CH0: max/min/mean/積算  CH1: 〃   │
├───────────────────────────────────────────┤
│ [時系列グラフ] ch0_A / ch1_A 折れ線(Plotly)  │
├───────────────────────────────────────────┤
│ [補助グラフ(任意)] rssi 推移                  │
└───────────────────────────────────────────┘
```

- CH0=青系、CH1=橙系など判別しやすい配色。凡例クリックで表示ON/OFF(Plotly標準機能)。
- ドロップゾーンはドラッグオーバー中に枠色を変えるなど視覚フィードバックを付ける。
- データ未読込時は「CSVを読み込んでください」のプレースホルダを表示。

## 5. 実装ステップ(この順で作る)

1. **雛形作成**: `index.html` に HTML骨格 + CDN2本の `<script>` + 基本CSS。
2. **ドロップゾーンUI**: dragover/dragleave/drop と file input のイベント配線。
   受け取った FileList を次段へ渡す(まだ console.log で確認)。
3. **パーサ実装**: PapaParse で1ファイルをレコード配列化する関数
   `parseCsvFile(file): Promise<Record[]>` を作る。§2 の日時パース・JSON展開・
   スキップ件数を実装。1ファイルで件数が想定通りか console で検証。
4. **マージ&ソート**: 複数ファイル結果を累積し `t` 昇順ソートする状態管理を実装。
   ファイル一覧・総件数・スキップ件数をUIに反映。
5. **時系列グラフ**: ダウンサンプリング関数 → Plotly `scattergl` 2トレース描画。
   `Plotly.react` で再描画。X軸は日時、Y軸は「電流 [A]」。
6. **統計サマリ**: 全期間で max/min/mean/積算 を計算して表示。
7. **期間連動**: `plotly_relayout` を購読し、表示X軸範囲で統計を再計算・更新。
   `xaxis.autorange` へ戻ったら全期間で再計算。
8. **補助表示(任意・余力があれば)**: rssi の折れ線、NTP同期状況。
9. **仕上げ**: クリアボタン、空状態表示、エラー時のガード、軽いスタイリング。

## 6. 検証手順(実装後に必ず実施)

プロジェクト直下に実データのサンプルCSVが5つある
(`20260701_164316_847.csv` 等)。これらで動作確認する。

1. `index.html` をブラウザで開く。コンソールにエラーが出ないこと。
2. CSVを1つD&D → グラフと統計が表示される。件数が「行数-1(ヘッダー)」付近か確認。
3. 5つまとめてD&D → 受信日時順に連続した1本の時系列になっていること
   (期間: 2026/07/01 16:43 〜 2026/07/03 02:01 付近)。
4. グラフをドラッグズーム → 統計値(max/min/mean/積算)がその範囲で更新されること。
5. 凡例クリックで CH0/CH1 の表示ON/OFFが効くこと。
6. 値の妥当性: ch0_A の最大が約47、ch1_A の最大が約43 前後に収まること(§2.4)。
7. 「クリア」で全消去 → 空状態表示に戻ること。

※検証は可能ならブラウザプレビュー機能で実際に開いて目視確認する。
   スクリーンショットを撮り、グラフが描画されていることを確認すること。

## 7. 実装上の注意(ハマりどころ)

- **エスケープ二重処理禁止**: PapaParse通過後のペイロードはそのまま `JSON.parse` する
  (自前で `""`→`"` 変換しない)。§2.1。
- **日時は自前パース**: `new Date(文字列)` に頼らない。§2.2。
- **BOM**: ヘッダー名で列判定しない。列インデックスで扱う。§2.1。
- **積算のファイル境界**: 大きな `dt` を積算に入れない(閾値60秒でスキップ)。§3.3。
- **性能**: 描画は必ず間引き+`scattergl`。統計は全点で計算。§3.2。
- **累積マージ**: 追加D&Dは置き換えでなく追記。クリアボタンで全消去。§3.1。
