# 実装手順書：グラフ表示粒度を約5倍に細かくする

## 目的
時系列グラフをもっと細かい粒度で確認できるようにする。
現状は1チャンネルあたり最大 5,000 点に間引かれており、これがグラフの細かさの上限になっている。
これを **約5倍（25,000 点）** に引き上げ、より細部まで見えるようにする。

対象は **通常版・リアルタイム版の両方**。

---

## 背景（変更箇所の仕組み）
両ファイルに `downsample()` 関数があり、系列全体を `MAX_POINTS` 点まで間引いている。

```js
function downsample(arr) {
  const n = arr.length;
  if (n <= MAX_POINTS) return arr;
  const step = Math.ceil(n / MAX_POINTS);   // ← MAX_POINTS が大きいほど step が小さくなり、細かくなる
  const out = [];
  for (let i = 0; i < n; i += step) out.push(arr[i]);
  if (out[out.length - 1] !== arr[n - 1]) out.push(arr[n - 1]);
  return out;
}
```

`MAX_POINTS` を 5000 → 25000 にすると間引きの `step` が約 1/5 になり、プロットされる点数が最大約5倍になる = 粒度が約5倍細かくなる。
グラフは `scattergl`（WebGL描画）なので、1系列25,000点程度なら描画性能は問題ない。

1. `html/index.html`（通常版：CSV手動読み込み）
2. `html/realtime/index.html`（リアルタイム版）

## 変更内容

### 1. `html/index.html`
`<script>` 冒頭付近（281行目あたり）の定数を変更する。

- 変更前
  ```js
  const MAX_POINTS = 5000;
  ```
- 変更後
  ```js
  const MAX_POINTS = 25000; // 表示粒度を約5倍に（1チャンネルあたり最大プロット点数）
  ```

### 2. `html/realtime/index.html`
同じく `<script>` 冒頭付近（269行目あたり）の定数を変更する。

- 変更前
  ```js
  const MAX_POINTS = 5000;
  ```
- 変更後
  ```js
  const MAX_POINTS = 25000; // 表示粒度を約5倍に（1チャンネルあたり最大プロット点数）
  ```

> 注意：両ファイルとも `MAX_POINTS` の定義は各ファイルに **1箇所ずつ** しかない。他の 5000 という数値（もし存在しても）は変更しないこと。`downsample()` 関数本体・その他のロジックは一切変更しない。

## 変更してはいけないもの（スコープ外）
- `downsample()` 関数の実装ロジック
- 統計計算（`computeChannelStats` など）
- 期間指定・ライブ追従（`setRange` / `refreshView`）まわり
- `INTEGRAL_GAP_THRESHOLD_MS` などその他の定数

今回はあくまで `MAX_POINTS` の値のみを 5000 → 25000 に変更する。

---

## 動作確認

### 通常版（`html/index.html`）
1. ブラウザで開く（Dockerで配信中なら `http://localhost:8081/`）。
2. 十分に点数の多いCSV（25,000点を超えるもの、または複数ファイル）を読み込む。
3. グラフを拡大（ドラッグでズーム）し、変更前よりも線が細かく（点が密に）描かれることを確認する。
4. 描画がカクつかない／固まらないことを確認する。

### リアルタイム版（`html/realtime/index.html`）
1. `http://localhost:8081/realtime/` を開く。
2. データが蓄積された状態でグラフの細かさが向上していることを確認する。
3. ライブ追従ON/OFF、ポーリング更新が従来どおり動作すること（エラーにならないこと）を確認する。

### 共通の確認観点
- ブラウザのコンソールにエラーが出ていないこと。
- 点数が少ないデータ（25,000点未満）では従来どおり全点表示され、見た目が変わらないこと。

---

## 補足（今後さらに粒度を変えたい場合）
粒度は `MAX_POINTS` の1値で決まる。さらに細かく／粗くしたい場合はこの定数のみを増減すればよい（例：50000 でさらに倍）。ただし極端に大きくすると描画・メモリ負荷が上がるため、実データ量に応じて調整すること。
