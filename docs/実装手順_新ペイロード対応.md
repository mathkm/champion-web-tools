> 新定義は「1行 = 1センサ 1値」。この構造差の吸収が本改修の核心。

## 2. 旧定義（現行）ペイロード仕様

CSV 1行の例：

```
2026/07/03 00:00:00.362,test/volt0,"{""is_ntp_synced"":false,""timestamp"":110999751,""ch0_A"":27.34,""ch1_A"":17.03,""uptime_s"":110999,""rssi"":-57}"
```

- 1列目 `受信日時`：`YYYY/MM/DD HH:MM:SS.mmm`（ローカル時刻。これを時間軸に使う）
- 3列目 JSON の主なキー：`ch0_A`（CH0 電流[A]）, `ch1_A`（CH1 電流[A]）, `rssi`, `is_ntp_synced`, `timestamp`(epoch ms), `uptime_s` ほか診断値
- **1行に両チャンネルが入る**。

## 3. 新定義ペイロード仕様（machine.md）

CSV 1行の例（3列目 JSON が machine.md 形式に変わる。外枠は同じ想定）：

```
2026/07/03 00:00:00.362,test/volt0,"{""mac"":""AA:BB:CC:DD:EE:01"",""ip"":""192.168.0.11"",""timestamp"":""2026-07-03T00:00:00.000+09:00"",""sensor_type"":""1"",""sensor_no"":1,""value"":""27.34""}"
```

- 3列目 JSON のキー：`mac`, `ip`, `timestamp`(ISO8601 文字列), `sensor_type`(文字列), `sensor_no`(数値), `value`(**文字列**。内容は sensor_type 依存)
- `sensor_type="1"` のとき `value` は電流値（数値文字列）。`Number(value)` で数値化する。
- **1行 = 1センサ 1値**。CH0 と CH1 は**別々の行**として届く（`sensor_no` で判別）。

---

## 4. 変更方針

### 4.1 フォーマット判定（行単位）

3列目を `JSON.parse` した後、**キーの有無で判定**する：

- `ch0_A` または `ch1_A` を持つ → **旧定義**
- `sensor_type` と `value` を持つ → **新定義**
- どちらでもない → スキップ（`skipped++`）

トピック列（2列目）には依存しない（現行同様）。

### 4.2 内部データモデルの変更（本改修の要）

現行は「1レコード＝両チャンネル」：`records = [{ t, ch0, ch1, rssi, ntp }]`。
新定義は1行1チャンネルのため、この対を作れない。**チャンネル別の独立時系列**に変更する：

```js
// 各チャンネルは独立した時系列サンプルの配列
// sample = { t: Date, v: number }
let series = { ch0: [], ch1: [] };
```

- **旧定義行**：`ch0_A`/`ch1_A` から `series.ch0` と `series.ch1` の両方へ push（各値が有限のときのみ）。同一 `t` を共有。
- **新定義行**：`sensor_type==="1"` かつ `SENSOR_NO_TO_CH[sensor_no]` が引けるとき、`Number(value)` が有限なら該当チャンネルへ push。それ以外はスキップ。
- 各チャンネル配列は読み込み後に `t` で昇順ソート。
- `rssi` / `ntp` は**モデルから削除**（4.4）。

この変更により、統計・グラフ・積算は「チャンネルごとに、そのチャンネル自身のサンプル列で計算」に統一される（旧・新どちらの由来でも同じ下流処理で扱える）。

### 4.3 センサ→チャンネル対応（設定定数）

コード先頭（`MAX_POINTS` 付近）に置く：

```js
// sensor_type="1"（電流）のみ対象。sensor_no → 表示チャンネル。要確認(前提B)。
const CURRENT_SENSOR_TYPE = "1";
const SENSOR_NO_TO_CH = { 1: 'ch0', 2: 'ch1' };
```

### 4.4 RSSI 表示の削除

新定義に rssi が無いため、**RSSI パネルを完全撤去**する（旧データに rssi があっても表示しない）。削除対象：

- HTML：`<section class="panel">…補助情報:RSSI推移…</section>`（`#rssiEmpty` / `#rssiChart` を含むセクション）
- CSS：`#chart, #rssiChart { … }` の `#rssiChart`、および `#rssiChart { height: 220px; }`
- JS：DOM参照 `rssiEmpty` / `rssiChartDiv`、関数 `renderRssiChart()` 全体、`renderAll()` 内の `renderRssiChart()` 呼び出し
- モデル：`rssi`（および使っていない `ntp`）フィールド

パース時に旧データの `rssi` キーが存在しても**単に無視**する（エラーにしない）。

### 4.5 時刻の扱い

新旧とも1列目 `受信日時` を `parseDateTime()` で解釈して `t` にする（前提D）。既存の `parseDateTime` / `DATETIME_RE` はそのまま流用。新定義 payload の `timestamp`（ISO8601）用パーサは**追加不要**。

## 5. 具体的な修正手順（index.html の該当箇所）

> 行番号は現行 `index.html` 基準の目安。関数名で特定すること。

### 手順1：設定定数の追加（`(function(){ "use strict";` 直後、`MAX_POINTS` 付近）
4.3 の `CURRENT_SENSOR_TYPE` と `SENSOR_NO_TO_CH` を追加。

### 手順2：状態変数の置換（現 `let records = []` 付近）
`records` を廃止し `let series = { ch0: [], ch1: [] };` を導入。
`loadedFiles` / `totalSkipped` はそのまま。

### 手順3：パーサの改修（`parseCsvFile` 内のループ）
行ごとに：
1. `t = parseDateTime(row[0])`。null ならスキップ（ヘッダ行/不正行。BOM 付き先頭行も従来通りここで弾かれる）。
2. `payload = JSON.parse(row[2])`（失敗時スキップ）。
3. 判定と格納：

```js
let usedThisRow = false;
if ('ch0_A' in payload || 'ch1_A' in payload) {
  // 旧定義
  const ch0 = Number(payload.ch0_A);
  const ch1 = Number(payload.ch1_A);
  if (Number.isFinite(ch0)) { out.ch0.push({ t, v: ch0 }); usedThisRow = true; }
  if (Number.isFinite(ch1)) { out.ch1.push({ t, v: ch1 }); usedThisRow = true; }
} else if ('sensor_type' in payload && 'value' in payload) {
  // 新定義（1行1値）
  if (String(payload.sensor_type) === CURRENT_SENSOR_TYPE) {
    const ch = SENSOR_NO_TO_CH[payload.sensor_no];
    const v = Number(payload.value);
    if (ch && Number.isFinite(v)) { out[ch].push({ t, v }); usedThisRow = true; }
  }
}
if (!usedThisRow) skipped++;
```

`parseCsvFile` は各チャンネルのサンプル配列（例 `{ ch0:[], ch1:[] }`）と `skipped`、および
「読込点数（= 採用サンプル数 = ch0数 + ch1数、または採用行数）」を返す形に変える。
**「読込点数」バッジの意味を新旧で揃える**こと（推奨：採用サンプル総数 `ch0.length + ch1.length`）。

### 手順4：マージ／ソート（`handleFiles`）
各ファイル結果を `series.ch0` / `series.ch1` に連結。全読み込み後に**チャンネルごとに** `t` 昇順ソート：
```js
series.ch0.sort((a,b)=>a.t-b.t);
series.ch1.sort((a,b)=>a.t-b.t);
```
`clearAll` は `series = { ch0: [], ch1: [] }` に戻す。

### 手順5：時間範囲ヘルパの追加
`records[0].t` / `records[last].t` に依存していた箇所を、2チャンネル横断の最小・最大時刻に置換：
```js
function allSamples() { return series.ch0.concat(series.ch1); }
function hasData() { return series.ch0.length > 0 || series.ch1.length > 0; }
function dataStartT() { /* ch0,ch1 の先頭のうち小さい方。両空なら null */ }
function dataEndT()   { /* ch0,ch1 の末尾のうち大きい方。両空なら null */ }
```
`setRange` / プリセット / 入力欄ハンドラ内の `records[0].t`・`records[records.length-1].t`・`records.length===0`
を上記ヘルパに置換する。

### 手順6：グラフ描画（`renderChart`）
`downsample(series.ch0)` と `downsample(series.ch1)` から2トレースを作る（各 `x: s.map(r=>r.t)`, `y: s.map(r=>r.v)`）。
空判定は `hasData()`。トレース名・色は現行踏襲（ch0=青 `#1f77b4`, ch1=橙 `#ff7f0e`）。

### 手順7：統計（`computeStats` / `renderStats`）
`computeStats` を**チャンネル別配列**を受け取る形に変更（max/min/mean/積算をそのチャンネルの連続サンプルで計算）。
積算は現行同様、隣接サンプル間 `dt` が `INTEGRAL_GAP_THRESHOLD_MS`(60秒) 以内のときのみ台形則で加算。
`renderStats(startMs, endMs)` は各チャンネルを範囲フィルタしてから `computeStats` に渡す。
`データ点数` は各チャンネルの範囲内サンプル数を表示（CH0/CH1 で異なってよい）。
表示期間ラベルは範囲内サンプルの先頭〜末尾時刻（両チャンネル横断）で作る。

### 手順8：範囲選択（`setRange`）
内部を「startMs/endMs を決める → 各チャンネルを範囲フィルタ → `renderStats` → 必要なら `Plotly.relayout`」に整理。
`suppressRelayout` とグラフ双方向連動（`onChartRelayout`）のロジックは現行のまま維持。

### 手順9：RSSI 撤去（4.4）
HTML セクション・CSS・`renderRssiChart`・`renderAll` 内呼び出し・DOM参照・モデルの `rssi`/`ntp` を削除。

### 手順10：`renderFileInfo` のバッジ
`読込点数` を新モデルの採用サンプル総数に合わせる（手順3の定義と一致させる）。

## 7. エッジケースの扱い

- `sensor_type` が `"1"` 以外（例 `"2"` RS-232C）：スキップ（`skipped++`）。エラーにしない。
- `SENSOR_NO_TO_CH` に無い `sensor_no`：スキップ。
- `value` が数値化不能（`Number(value)` が非有限）：スキップ。
- `sensor_no` が数値でなく文字列で来る場合に備え、必要なら `Number(payload.sensor_no)` で正規化してからマップ引き。
- ヘッダ行・空行・BOM 付き先頭行：`parseDateTime` が null を返すことで従来通り自動スキップ。
- 片チャンネルしか届かない期間：そのチャンネルのみ描画・統計され、他方は空でも破綻しないこと。
