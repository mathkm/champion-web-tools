# 実装手順書：読み込みファイル一覧の表示改善

## 0. この手順書について

大量データ読み込み時にファイル数が増え、`data/` 読込ファイル一覧の縦スクロールが煩雑になる。
一覧部分（`#fileInfo` のテーブル）を以下の2点に沿って改善する。

- 対象ファイル：`C:\dev\ampereVisualizer\index.html`（**このファイルのみ**を修正）
- 実装担当：Sonnet
- 変更範囲：一覧テーブルの**表示順**と、一覧領域の**高さ制限＋スクロール**のみ。読込・集計・グラフ・統計ロジックには一切手を入れない。

---

## 1. 変更要件

| # | 要件 | 補足 |
|---|---|---|
| 1 | 一覧の表示順を**ファイル名の自然ソート（natural sort）で降順**にする。 | 例：`20260708_084132_138.csv` が `20260703_000000_362.csv` より上（新しい名前が上）。数字を数値として比較する自然順（`file2` < `file10`）。単純な文字列比較にしない。 |
| 2 | 一覧の縦幅が **150px 以上になる場合はスクロール**させる。 | 150px 未満のときはスクロールバーを出さず、内容の高さに従う。150px を超えたら 150px で頭打ちにし、縦スクロールで残りを見せる。 |

---

## 2. 現状

- CSS：`#fileInfo`（`index.html` 内 `<style>`、`margin-top:14px; font-size:13px;` のブロック）に高さ制限なし。中に `#fileInfo table`（`width:100%; border-collapse:collapse;`）。
- JS：`renderFileInfo()` が `loadedFiles` を**挿入順**（`for (const f of loadedFiles)`）でそのまま `<tr>` に展開している。
- `loadedFiles` は `[{ name, rowCount, skipped }]`。**表示専用**で、読込データ（`series`）の正しさとは独立（`series` は時刻で別途ソート済み）。

---

## 3. 具体的な修正手順

### 手順1：自然ソート用コンパレータを定義（先頭の定数付近）

`(function () { "use strict";` 直後の定数群（`MAX_POINTS` / `SENSOR_NO_TO_CH` 付近）に、`Intl.Collator` を1つだけ生成して使い回す：

```js
// ファイル名の自然順ソート用（数字を数値として比較。file2 < file10）
const NAME_COLLATOR = new Intl.Collator(undefined, { numeric: true, sensitivity: 'base' });
```

> 毎回 `new` せず1回だけ生成する（描画のたびに作らない）。

### 手順2：`renderFileInfo()` の描画順を降順自然ソートに変更

`renderFileInfo()` 内、テーブル本体を組み立てるループの**直前**で、`loadedFiles` の**コピー**を降順ソートし、それを回す。**`loadedFiles` 本体は破壊的ソートしない**（表示専用の並べ替えに留め、他への副作用を避ける）。

変更前：

```js
    let html = '<table><thead><tr><th>ファイル名</th><th>読込点数</th><th>スキップ</th></tr></thead><tbody>';
    for (const f of loadedFiles) {
      html += `<tr><td>${escapeHtml(f.name)}</td><td>${f.rowCount.toLocaleString()}</td><td>${f.skipped.toLocaleString()}</td></tr>`;
    }
    html += '</tbody></table>';
    fileInfo.innerHTML = html;
```

変更後：

```js
    // 表示専用に、ファイル名の自然順で降順ソートしたコピーを使う（loadedFiles本体は不変）
    const displayFiles = loadedFiles.slice().sort((a, b) => NAME_COLLATOR.compare(b.name, a.name));

    let html = '<table><thead><tr><th>ファイル名</th><th>読込点数</th><th>スキップ</th></tr></thead><tbody>';
    for (const f of displayFiles) {
      html += `<tr><td>${escapeHtml(f.name)}</td><td>${f.rowCount.toLocaleString()}</td><td>${f.skipped.toLocaleString()}</td></tr>`;
    }
    html += '</tbody></table>';
    fileInfo.innerHTML = html;
```

> `NAME_COLLATOR.compare(b.name, a.name)`（b と a を入れ替え）で**降順**になる。昇順にしたい場合は `(a.name, b.name)`。

### 手順3：一覧領域に高さ制限＋スクロールを付与（CSS）

`#fileInfo` のルールに `max-height` と `overflow-y` を追加する：

```css
  #fileInfo {
    margin-top: 14px;
    font-size: 13px;
    max-height: 150px;
    overflow-y: auto;
  }
```

- `max-height:150px` により内容が 150px を超えた時だけ頭打ちになり、`overflow-y:auto` で縦スクロールが出る（150px 未満ではスクロールバーは出ない）。要件2を満たす。

### 手順4（推奨）：スクロール時にヘッダ行を固定

スクロールすると `<thead>` が流れて列見出しが見えなくなるため、ヘッダを上部に固定する。`#fileInfo table` 関連の CSS に追記：

```css
  #fileInfo thead th {
    position: sticky;
    top: 0;
    background: #fff;
    z-index: 1;
  }
```

> これは要件外の UX 改善だが低リスク。含めることを推奨。含めない場合でも要件1・2は満たす。

---

## 4. 留意点・エッジケース

- **`loadedFiles` を破壊的にソートしない**（`.slice()` してからソート）。集計バッジ `ファイル: N`（`loadedFiles.length`）や他処理は件数のみ参照で順序非依存だが、副作用を避けるため表示専用コピーで並べ替える。
- ソート対象は `f.name`（表示名＝ファイル名）。同名が来た場合の相対順は問わない（実害なし）。
- 空状態（`loadedFiles.length === 0`）の分岐（`fileInfo.innerHTML=''`）はそのまま。
- `escapeHtml` によるエスケープは現状踏襲（変更しない）。

---

## 5. 検証手順

`.claude/launch.json` のサーバでローカル起動し、ブラウザで確認する。

1. **自然ソート降順**：`data/` の CSV を複数（できれば全件）ドロップし、一覧の1列目が
   `2026...` の新しい名前（大きい方）から降順で並ぶこと。`_9` と `_10` のような桁差で
   `_10` が `_9` より上（数値として比較されている）ことを、名前にそういう差がある場合に確認。
2. **150px スクロール**：
   - 数件だけ読み込み（高さ < 150px）→ スクロールバーが出ないこと。
   - 多数読み込み（高さ ≥ 150px）→ 一覧が 150px で頭打ちになり縦スクロールが出ること。ページ全体が
     一覧のせいで無限に伸びないこと。
3. **ヘッダ固定（手順4を入れた場合）**：一覧内をスクロールしても列見出し（ファイル名/読込点数/スキップ）が
   上部に残ること。
4. グラフ・統計・期間選択・スキップ計上など**他機能に回帰がない**こと。コンソールにエラーが出ないこと。
