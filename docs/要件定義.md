# ampereVisualizer 要件定義書

作成日: 2026-07-03

## 1. 背景・目的
MQTT経由で収集した電流ロガーのCSVログを可視化し、2チャンネルの電流値
(`ch0_A` / `ch1_A`)の時系列変化と統計を、ブラウザ上で手軽に確認できるように
する。CSVファイルは今後も継続的に増えていくため、固定ファイルを埋め込むのでは
なく、**ドラッグ&ドロップで任意のCSVを読み込める汎用ビューア**とする。

## 2. 入力データ仕様(実データ調査結果)

| 項目 | 内容 |
|---|---|
| ファイル名形式 | `YYYYMMDD_HHMMSS_mmm.csv`(記録開始時刻と推測) |
| 列構成 | `受信日時`, `トピック`, `ペイロード`(3列) |
| 受信日時形式 | `YYYY/MM/DD HH:MM:SS.mmm`(ミリ秒精度) |
| トピック | `test/volt0`(現状単一) |
| 文字コード | UTF-8 with BOM |
| サンプリング間隔 | 約1.4秒 |
| 参考:調査時点の総行数 | 約70,836行 / 5ファイル / 約1.4日分 |

### ペイロード(JSON)フィールド
| フィールド | 意味 | 単位 | 実測レンジ |
|---|---|---|---|
| `ch0_A` | 電流 チャンネル0 | A | 0.245 〜 46.97 |
| `ch1_A` | 電流 チャンネル1 | A | 0.249 〜 43.40 |
| `timestamp` | 起動からの経過時間 | ms | — |
| `uptime_s` | 起動からの経過時間 | s | — |
| `rssi` | WiFi電波強度 | dBm | -80 〜 -48 |
| `is_ntp_synced` | NTP同期状態 | bool | 現状全て false |

## 3. 成果物の形態
- **クライアント完結型のWebダッシュボード**(サーバー不要)。
- 配布・再利用のしやすさを優先し、**単体HTMLファイル**を基本とする
  (CSVパース・グラフ描画をブラウザ内で完結)。
- ダブルクリックまたはブラウザで開くだけで動作する。

## 4. 機能要件

### F1. データ読み込み(ドラッグ&ドロップ)
- CSVファイルをD&D、またはファイル選択で読み込む。**複数ファイル同時可**。
- 受信日時をパースし、ペイロードJSONを展開。
- 複数ファイルを**受信日時順に連結して1本の時系列**へ統合(欠損/重複はスキップ)。
- BOM・ダブルクォートエスケープ(`""`)を正しく処理。

### F2. 時系列グラフ(主表示)
- `ch0_A` / `ch1_A` を重ね描きした折れ線グラフ。
- インタラクティブなズーム/パン、ホバーで値表示。
- チャンネルの表示ON/OFF切替。
- 期間(時間範囲)の絞り込み。

### F3. 統計サマリ(併記)
- チャンネルごとに 最大 / 最小 / 平均 を表示。
- 消費電力量に相当する**積算値**(電流の時間積分 ≒ A·h)を表示。
- 対象は「表示中の期間」に連動して再計算。

### F3b. 統計期間の明示的な選択(2026-07-03 追加)
- 統計対象期間を UI から明示的に指定できる。
  - **日時指定**: 開始/終了の日時入力欄(秒単位)。
  - **プリセット**: 「全期間」「直近1時間」「直近6時間」「直近24時間」ボタン。
    「直近」の基準は現在時刻ではなく**読み込みデータの最終時刻**。
- 期間選択とグラフ表示範囲は**双方向に連動**する:
  - 期間を選択 → グラフのX軸もその範囲にズーム。
  - グラフをズーム/パン → 日時入力欄にも反映。
  - 常に「統計値・入力欄・グラフ表示範囲」が一致し、迷わない。

### F4. 補助情報(任意/優先度低)
- `rssi` の推移、`is_ntp_synced` の状態など通信品質の確認表示。
- データ点数・期間・読み込みファイル一覧の表示。

## 5. 非機能要件
- 数万〜十数万行規模でも実用的な描画速度(必要に応じ間引き/ダウンサンプリング)。
- 主要ブラウザ(Chrome/Edge)で動作。
- 外部送信なし(ローカル完結、プライバシー配慮)。

## 6. スコープ外(今回は対象としない)
- リアルタイム(ライブ)収集・MQTT直接購読。
- サーバー側でのデータ永続化・DB化。
- 複数トピック/複数デバイスの同時比較(将来拡張候補)。

## 7. 技術方針(案)
- CSVパース: PapaParse、グラフ描画: Plotly.js(いずれもCDNまたは埋め込み)。
- 大量データ対策として WebGL 描画(`scattergl`)・ダウンサンプリングを検討。

## 8. 確定した論点(ヒアリング結果)
- 形態: Webダッシュボード。
- 分析主眼: 時系列トレンドと統計の両方をバランスよく。
- ファイル統合: 全期間を連結して1本の時系列。ただしCSVは参考用で今後も
  増えるため、D&Dで都度読み込む方式とする。
